Who is Culturally Relevant Science (CRSci)?

Culturally Relevant Science is a nonprofit organization dedicated to creating innovative, affordable, and inclusive K–12 STEM curriculum for all students, especially those from underrepresented communities. They believe that every student, regardless of how they identify, deserves to see themselves represented in STEM, which helps increase engagement, achievement, and pursuit of STEM careers.

Project Type

Nonprofit redesign project completed as volunteer work through the Develop for Good program.

UX Designer

Problem Statement

CRSci provides culturally relevant STEM curriculum to K–12 schools, but all materials were stored in a large Google Drive folder. Teachers struggled to locate updated lesson files, and admins had no way to monitor student exam performance. The lack of structure created delays, inconsistencies, and extra work for staff.

We asked: How might we create a platform where teachers can quickly access curriculum materials and admins can easily track student performance?

UX Process

Research

We conducted user interviews and competitive analysis to understand teacher and admin needs, as well as how other educational platforms handle curriculum delivery.

User Interviews

Teacher Insights:
  • Ease of access is critical
  • Adaptability matters
  • Representation is valued
Feature Requests from Teachers:
  • Walkthrough/tutorial for new users
  • Feedback/comment sections for teachers to share suggestions
Admin Insights:
  • Data drives decisions
  • Simplicity over complexity
  • Account and content control is important
Feature Requests from Admins:
  • Visual usage reports
  • Notifications when new CRS content is published

Competitive Analysis

We reviewed indirect competitors such as Khan Academy, CK-12, Code.org, OER Commons, and others to evaluate:

  • Navigation & Layout – how intuitively users could find resources
  • Visual Design – clarity, engagement, and accessibility
  • Content Discovery – search and filtering capabilities
  • Resource Interaction – preview, download, and customization options
  • Security Measures – access controls and file protection

User Testing

We tested prototypes with CRSci stakeholders. Feedback included the need to make file types easier to scan, improve unit labels, and simplify the analytics page. We iterated on spacing, hierarchy, naming, and the overall organization of the resource pages.

Final Design Highlights

The final prototype established a cleaner information structure and more intuitive navigation. Curriculum is now organized by grade and unit, reducing search friction for teachers, while admins gain a dedicated dashboard for monitoring student performance.

  • Curriculum library with filtering + preview modal
  • Unit pages with downloadable resources
  • Admin dashboard with analytics + class performance tracking
  • Consistent UI theme using CRSci brand colors and typography

Results / Outcome

The redesign improved clarity, reduced search time for lesson resources, and gave CRSci a foundation for long-term growth. Our stakeholder reviews confirmed that the new structure is more intuitive for teachers and provides admins with the visibility they previously lacked.

This project strengthened my communication skills, taught me the value of consistent components, and helped me prioritize a large problem space within realistic constraints.
